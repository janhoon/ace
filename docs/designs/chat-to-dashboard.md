## Vision

### 10x Check
The 10x version is an autonomous SRE agent: Ace doesn't just show you problems, it investigates anomalies, correlates across services, proposes fixes with runbooks, and manages your incident response. Chat-to-dashboard is the foundation — it establishes that the chat IS the interface, and the DashboardSpec becomes the portable primitive for all future AI-generated artifacts.

### Dream State (12 months)
The copilot becomes the primary way people interact with their monitoring stack. Manual dashboard creation feels as archaic as writing SQL by hand. Ace is known in the open source community as "the AI-native Grafana alternative" — not because it replaces Grafana feature-for-feature, but because it rethinks what a monitoring tool looks like when AI is the primary interface.

## Accepted Scope (added to this plan)

### 1. Sample Prompt Suggestions
- Replace the empty chat state in `CopilotPanel.vue` (line 434-437) with 3 clickable prompt suggestions
- Suggestions: "Show me p99 latency for my API endpoints", "Create an overview dashboard", "What's my error rate?"
- Clicking a suggestion fills the input and triggers `handleSend()`
- Only shown when `datasourceType === 'victoriametrics'` (matching tool-call gate)

### 2. Query Dry-Run Before Save
- After spec is generated and before save, automatically test each panel's `expr` against the datasource
- Use existing `queryDataSource()` from `api/datasources.ts` with a short time range
- Show per-panel status in the preview: green checkmark (has data), orange warning (empty/no data), red X (query error)
- Save button still works regardless of dry-run results — it's informational, not blocking
- Adds ~20 lines to `DashboardSpecPreview.vue` and reuses existing API

### 3. Demo Mode
- In `github_copilot.go`, extend the victoriametrics system prompt: if the AI's get_metrics call returns no metrics, instruct it to generate a dashboard using common example metrics (http_requests_total, node_cpu_seconds_total, process_resident_memory_bytes)
- In `DashboardSpecPreview.vue`, show a "Demo dashboard" badge when panels contain example metric names
- Badge text: "Demo dashboard — connect a real datasource to see your data"
- Badge style: `text-amber-500 bg-amber-500/10 rounded px-2 py-1 text-xs`
