# Security Policy

## Supported Versions

Security fixes are provided for the current `master` branch.

## Reporting a Vulnerability

Please do not report security issues in public GitHub issues.

Use GitHub private vulnerability reporting in this repository:

1. Open the repository Security tab.
2. Select Advisories.
3. Select Report a vulnerability.

Include a clear description, affected components, impact, and reproduction steps.

## Response Expectations

- Initial triage acknowledgement target: 3 business days.
- Status update target: 7 business days.
- Critical issues are prioritized for immediate remediation.
