The 7 Views

Overview of each screen's purpose and key components

1

Dashboards Explorer

Route: /app/dashboards

Browse, search, and organize dashboards. Card grid with live metric previews (sparklines, key stats). Folder hierarchy. "New Dashboard" button opens choice: blank or AI-generated. Search with filters (folder, tag, data source).

Replaces: DashboardsView + DashboardList

2

Dynamic Dashboard (Detail)

Route: /app/dashboards/:id

The dashboard itself. Draggable grid of panels (line charts, gauges, tables, stats, heatmaps, traces). Time range picker top-right. Edit mode toggle. Panel types: stat, line, bar, pie, gauge, table, log viewer, trace timeline. Inline AI insights — anomaly badges on panels, AI-suggested correlations between metrics. Panel edit modal for query building.

Replaces: DashboardDetailView

3

Services Overview

Route: /app/services

High-level service health at a glance. Service cards showing status (healthy/degraded/down), key metrics (latency, error rate, throughput), and dependency map. Click into a service to see its related dashboards and alerts. This is the "flight deck" — dense, scannable, monospace metrics everywhere.

New view — scaffold without backend wiring

4

Alert & Incident Explorer

Route: /app/alerts

All alerts in a dense, filterable table. Columns: severity, name, source, status (firing/resolved/silenced), duration, last triggered. Expandable rows for alert detail + history. Firing alerts surface AI-generated root cause suggestions inline. Incident grouping — related alerts clustered together. Silence/acknowledge actions.

Replaces: AlertsView

5

Explore (Metrics / Logs / Traces)

Route: /app/explore/:type

Unified explore view with sub-nav tabs (Metrics, Logs, Traces). Query builder with Monaco editor, data source selector, time range. Results below: line charts for metrics, log stream for logs, trace waterfall + service graph for traces. AI assists: natural language → query translation, anomaly highlighting in results.

Replaces: Explore, ExploreLogs, ExploreTraces

6

Dashboard Generation Flow

Route: /app/dashboards/new/ai

Guided AI dashboard creation. Step 1: Describe what you want to monitor (rich text input with suggestions). Step 2: AI generates spec — shows preview of proposed panels, data sources, layout. Step 3: User reviews, adjusts, confirms. Step 4: Dashboard materializes with generating animation. Also accessible via Cmd+K shortcut.

Enhances: existing chat-to-dashboard feature

7

Settings

Route: /app/settings/:section

Vertical tab layout (left nav within content area). Sections: General (org name, branding), Members, Groups & Permissions, Data Sources (moved here from top nav), AI Configuration, SSO/Auth. Dense forms, tonal section separators, monospace for config values.

Replaces: OrganizationSettings + DataSourceSettings
