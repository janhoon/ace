Sidebar Structure

What layout pattern fits Ace best?

A

Icon Rail + Flyout

Narrow icon strip (48-56px) always visible. Hovering or clicking expands a flyout panel with labels and sub-navigation. Maximizes content space. Think: VS Code, Linear, Figma.

B

Collapsible Sidebar

Toggle between a slim icon-only mode (~56px) and a full expanded mode (~220px). User controls when to expand/collapse. Similar to current behavior but with a proper collapsed state instead of fully hiding. Think: Grafana, GitHub.

C

Fixed Full Sidebar

Always-visible sidebar at ~200-240px. No collapse, no toggle. Clean and simple — the content area adjusts. Relies on the 1280px+ viewport requirement you already enforce. Think: Datadog, Notion.

D

Two-Level Sidebar

Narrow primary rail with top-level sections, plus a secondary panel that slides out to show sub-navigation for the selected section. Good for deep navigation trees. Think: Azure Portal, Jira.
